## 1. 项目愿景与定位

**项目名称**: DB Client

**一句话定位**: 企业级跨平台桌面数据库客户端，为数据库管理员、开发工程师和数据分析师提供统一、安全、高效的数据库操作体验。

**愿景陈述**: DB Client 旨在成为数据库从业者的「瑞士军刀」——在单一桌面应用中覆盖从连接管理到查询分析、从数据编辑到迁移导出的完整工作流，同时以 AES-256-GCM 加密和审计日志保障企业级安全合规需求。区别于 Navicat 的商业闭源、DBeaver 的 Java 重依赖，DB Client 以 Go 原生编译 + Wails 轻量前端实现零依赖分发、秒级启动、低内存占用，并在 PolarDB/GaussDB 等国产云数据库上提供比竞品更深入的适配支持。

**核心差异化价值**:

| 维度 | DB Client | Navicat | DBeaver |
|------|-----------|---------|---------|
| 分发体积 | ~15MB 单可执行文件 | ~200MB 安装包 | ~300MB (需 JVM) |
| 启动时间 | <2s | ~5s | ~8s |
| 内存占用 | ~50MB 基线 | ~150MB | ~300MB |
| 国产云数据库 | PolarDB/GaussDB 深度适配 | 仅基础支持 | 仅基础支持 |
| 密码存储安全 | AES-256-GCM 加密 | 明文/弱加密 | 明文配置文件 |
| 代码可控性 | 开源 Community 版 | 闭源商业 | 开源 LGPL |
| 依赖要求 | 零外部依赖 | 无 | 需要 JDK/JRE |

### 5.2 各版本里程碑详情

#### v1.0 — MVP (当前版本)

- **发布时间**: 2024 Q1 (已发布 v1.0.0 + v1.0.1 补丁)
- **目标**: 核心功能可用，基础稳定性
- **已交付功能**:
  - 6 种数据库类型连接 (PostgreSQL, MySQL, SQLite, Redis, PolarDB, GaussDB)
  - Monaco Editor SQL 编辑器 (语法高亮、多标签页)
  - 多查询批量执行 (`ExecuteMultiQuery`)
  - 表/视图/函数/索引/外键/统计信息查看
  - AES-256-GCM 密码加密存储
  - SQL 注入防护 (`sanitizeIdentifier`)
  - 连接池 (最大 50 连接，3s ping 健康检查)
  - 连接配置保存/加载/测试
  - 查询历史记录
  - i18n 中英双语支持
- **已知关键 BUG**:
  - BUG-001: PostgreSQL 连接池缓存键不一致导致数据库切换问题 (`pool.go:35`)
  - BUG-002: 缺少查询超时控制，长时间查询可能阻塞 UI (`query.go`)
- **验收标准**: 能成功连接 6 种数据库并执行基础 CRUD 查询

#### v1.5 — 稳定性提升

- **发布时间**: 2024 Q2
- **目标**: 修复关键问题，提升用户体验，补齐高频需求功能
- **计划交付**:
  - 查询超时控制 (默认 30s，可配置)
  - SQL 自动补全 (表名/字段名/关键字)
  - SQL 格式化 (已有 `sql_formatter.go` 基础)
  - 数据行内编辑 (INSERT/UPDATE/DELETE)
  - 操作日志完善
  - 连接分组管理 + 标签搜索
  - BUG-001/BUG-002 修复
- **验收标准**: 单次查询超时自动终止；自动补全响应 <200ms；数据编辑零数据丢失

#### v2.0 — 企业版

- **发布时间**: 2024 Q3
- **目标**: 满足企业级安全合规与团队协作需求
- **计划交付**:
  - 完整审计日志系统 (操作记录 + 安全追踪 + 合规报告)
  - 数据导入导出向导 (CSV/JSON/Excel)
  - 数据对比工具
  - SSH 隧道连接
  - 性能监控仪表盘 (实时查询、慢查询分析)
  - 敏感数据脱敏
  - 危险操作二次确认 (DROP/TRUNCATE)
  - 团队协作基础 (连接配置共享)
- **验收标准**: 审计日志覆盖全部操作；SSH 隧道连接 PostgreSQL 成功；导出 10万行数据 <30s

#### v3.0 — 高级版

- **发布时间**: 2024 Q4
- **目标**: 差异化竞争，扩展生态
- **计划交付**:
  - 跨数据库数据迁移
  - 增量数据同步
  - 插件系统架构
  - 云数据库深度支持 (阿里云 PolarDB-X / 腾讯云 TDSQL / AWS RDS)
  - NoSQL 支持 (MongoDB / Elasticsearch)
  - 数据可视化图表 + 自定义报表
  - 定时查询执行 + 任务调度
  - 版本控制集成 (Git)
- **验收标准**: 跨数据库迁移 100万行数据 <5min；插件加载 <1s；报表导出 PDF 格式完整

## 7. 风险评估

### 7.1 Top 5 风险与缓解策略

| # | 风险 | 严重性 | 概率 | 影响 | 缓解策略 |
|---|------|--------|------|------|---------|
| **R1** | CGO 编译依赖导致跨平台分发困难 | 高 | 中 | SQLite 驱动 (`go-sqlite3`) 需要 C 编译器，Linux/Mac 用户可能缺少 GCC；Windows 需 MinGW | (1) 构建脚本预装 CGO 工具链 (2) 评估替换为纯 Go SQLite 髹动如 `modernc.org/sqlite` (3) CI 环境固化编译配置 |
| **R2** | 长时间查询阻塞 UI 导致用户强制关闭应用 | 高 | 高 | Wails IPC 是同步调用，Go 侧长时间查询阻塞 WebView 主线程 | (1) v1.5 优先实现查询超时控制 (`query_timeout.go` 已有基础) (2) 异步查询执行 + 进度回调 (3) 默认超时 30s + 可配置 |
| **R3** | 连接池缓存键不一致导致查询错误的数据库 | 高 | 中 | `buildKey()` 与 `schema.go` 中键格式不匹配，用户切换数据库后查询落入旧库 | (1) BUG-001 已定位 (`pool.go:35`, `schema.go:45`) (2) v1.5 首个修复目标 (3) 统一键格式为 `{type}:{host}:{port}:{username}:{database}` |
| **R4** | Monaco Editor 大数据集渲染性能瓶颈 | 中 | 高 | 查询结果 >1000 行时 UI 卡顿 (BUG-007)，Monaco DOM 操作重 | (1) 结果表格与 Monaco 分离，表格用虚拟滚动 (2) 结果集上限 5000 行分页加载 (3) 后端流式返回替代全量返回 |
| **R5** | 单元测试覆盖率不足 (<30%) 导致回归风险 | 中 | 高 | 核心模块 (连接/查询/加密/池) 缺少系统测试，重构时容易引入缺陷 | (1) v1.5 补齐核心模块测试 (目标覆盖率 >60%) (2) CI 强制测试通过才能合并 (3) 优先覆盖 `pool.go`、`crypto.go`、`query.go` |
