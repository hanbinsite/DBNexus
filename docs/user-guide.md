## 安装

### Windows
1. 从 [Releases](https://github.com/hanbinsite/DBNexus/releases) 下载 `DBNexus-windows-amd64.zip`
2. 解压并运行 `DBNexus.exe`

### macOS
1. 下载 `DBNexus-darwin-amd64.tar.gz`
2. 解压并运行 `DBNexus`

### Linux
1. 下载 `DBNexus-linux-amd64.tar.gz`
2. 解压并运行 `./DBNexus`

### 从源码构建
```bash
go install github.com/wailsapp/wails/v2/cmd/wails@latest
git clone git@github.com:hanbinsite/DBNexus.git
cd DBNexus
wails build
```

## 连接数据库

### 新建连接
1. 点击左侧栏 **+ 新建连接** 按钮
2. 选择数据库类型 (PostgreSQL / MySQL / SQLite / Redis / PolarDB / GaussDB)
3. 填写连接信息:
   - **名称**: 连接显示名称
   - **主机**: 数据库服务器地址
   - **端口**: 数据库端口
   - **用户名 / 密码**: 认证信息
   - **数据库**: 默认数据库名
4. 点击 **测试连接** 验证
5. 点击 **保存** 存储

### SSH 隧道
1. 在连接对话框中勾选 **SSH 隧道**
2. 填写 SSH 服务器信息
3. 选择认证方式 (密码 / 密钥文件)

### SSL/TLS
1. 勾选 **SSL/TLS 加密连接**
2. 可选: 填写 CA 证书 / 客户端证书 / 客户端密钥路径
3. 选择最低 TLS 版本
4. 自签名证书可勾选 **跳过证书验证**

### 连接分组
- 右键连接 → **编辑连接** → 设置分组
- 在设置中管理连接分组

### 自动连接
- 编辑连接时勾选 **自动连接**
- 应用启动时自动连接

## SQL 查询

### 基本操作
1. 点击 **+ 新建标签页** (或 `Ctrl+T`)
2. 在 SQL 编辑器中输入查询
3. 按 `F5` 执行
4. 结果显示在下方结果面板

### 多语句执行
- 用分号 `;` 分隔多条 SQL
- 每条语句单独执行并显示结果

### 执行选中语句
- 选中部分 SQL 文本
- 按 `Ctrl+F5` 仅执行选中部分

### 查询历史
- 点击侧栏 **查询历史** 面板
- 搜索 / 筛选 (全部 / 慢查询 / 错误)
- 点击历史项加载到编辑器

### SQL 格式化
- 按 `Ctrl+Shift+F` 格式化当前 SQL

### EXPLAIN 分析
- 点击工具栏 **EXPLAIN** 按钮
- 查看查询执行计划
- AI 自动提供优化建议

## 数据编辑

### 查看表数据
1. 在数据库树中双击表名
2. 数据显示在数据视图面板
3. 支持分页 / 排序 / 筛选

### 编辑数据
1. 双击单元格进入编辑模式
2. 修改值后按 `Enter` 确认
3. 按 `Esc` 取消编辑

### 添加行
- 点击工具栏 **+ 添加行** 按钮
- 填写各列值
- 点击 **保存** 提交

### 删除行
- 选中行
- 点击 **删除行** 按钮
- 确认删除

### 批量编辑
- 选中多行
- 右键 → **批量编辑**
- 支持事务模式 (全部成功或全部回滚)

## 导入导出

### 导出
1. 选中表或执行查询
2. 点击工具栏 **导出** 按钮
3. 选择格式: CSV / JSON / Excel / SQL
4. 选择保存路径

### 导入
1. 选中目标表
2. 点击 **导入** 按钮
3. 选择文件 (CSV / JSON)
4. 映射列
5. 执行导入

### 流式导出
- 大数据集自动使用流式导出
- 避免内存溢出

## AI 助手

### 配置 AI
1. 打开 **设置** (`Ctrl+,`)
2. 进入 **AI** 部分
3. 选择 Provider:
   - **OpenAI Compatible**: 填写 API Key + Base URL
   - **Ollama**: 填写本地 Ollama 地址
4. 选择模型
5. 点击 **测试连接**

### AI 对话
1. 点击侧栏 **AI 助手** 面板
2. 输入问题 (自然语言)
3. AI 自动识别意图:
   - **查询**: 生成 SQL
   - **解释**: 解释 SQL 逻辑
   - **优化**: 提供优化建议
4. 点击 **插入编辑器** 使用生成的 SQL

### NL2SQL
- 点击工具栏 **NL2SQL** 按钮
- 输入自然语言描述
- AI 生成对应 SQL

### AI 优化
- 选中 SQL
- 点击工具栏 **AI 优化** 按钮
- AI 返回优化后的 SQL + 索引建议

### 自动索引建议
1. 选中表
2. 在 AI 面板点击 **分析索引**
3. 查看推荐索引列表
4. 点击 **创建索引** 生成 DDL

## Redis 浏览器

### 连接 Redis
1. 新建连接，类型选择 **Redis**
2. 填写主机和端口
3. 可选: 密码认证

### 浏览 Key
- 左侧显示数据库列表 (db0-db15)
- 点击数据库显示 Key 列表
- 搜索框支持模式匹配 (`user:*`)

### 操作 Key
- **查看**: 点击 Key 显示值
- **设置**: 编辑 Value 后保存
- **删除**: 右键 → 删除
- **TTL**: 查看/设置过期时间

### 安全
- 仅允许白名单内的 80+ 安全命令
- 危险命令 (FLUSHALL/CONFIG) 被禁用

- 点击侧栏 **性能** 面板

### 系统信息
- Go 版本 / Goroutine 数 / 内存使用 / GC 统计

- 总连接 / 活跃 / 空闲 / 最大连接

- 查看当前正在执行的查询
- 点击 **取消** 终止查询

## Git 集成

### 添加仓库
1. 点击 Git 面板 **+ 添加**
2. 选择本地 Git 仓库路径

### 查看变更
- **文件变更**: 显示已修改/新增/删除的文件
- **提交历史**: 显示最近提交记录

### 操作
- **Pull**: 拉取远程更新
- **Push**: 推送本地提交
- **Commit**: 输入提交信息并提交
- **新建分支**: 创建并切换到新分支

## 报表设计器

### 创建模板
1. 点击侧栏 **报表** 面板
2. 点击 **+ 新建模板**
3. 填写模板名称和描述

### 添加区块
- **表格**: SQL 查询结果以表格显示
- **图表**: 柱状图 / 折线图 / 饼图
- **摘要**: 聚合统计 (COUNT/SUM/AVG)
- **文本**: 静态文本

### 执行报表
1. 选择模板
2. 填写参数 (如有)
3. 点击 **执行报表**
4. 查看结果

## 插件系统

### 注册插件
1. 打开 **设置** → **插件**
2. 点击 **注册**
3. 填写插件名称 / 版本 / 类型

### 插件类型
- **QueryHook**: 查询执行前/后钩子
- **EditorHook**: 编辑器钩子
- **ConnectionHook**: 连接钩子
- **ExportHook**: 导出钩子
- **ImportHook**: 导入钩子
- **UIHook**: UI 扩展钩子
- **DataHook**: 数据操作钩子
- **SchemaHook**: Schema 变更钩子

### 原生插件
- Go plugin (.so 文件) 动态加载
- 放置在 `~/.db-client/plugins/native/` 目录
- 支持 Init/Cleanup 生命周期函数

## 常见问题

### Q: SQLite 连接失败?
A: SQLite 需要 CGO 支持。从源码构建时确保 `CGO_ENABLED=1`。

### Q: AI 功能不工作?
A: 检查设置中 AI Provider 配置。Ollama 需要本地运行 Ollama 服务。

### Q: 连接密码安全吗?
A: 密码使用 AES-256-GCM 加密存储在 `~/.db-client/connections.json`。

### Q: 如何贡献代码?
A: 阅读 [CONTRIBUTING.md](CONTRIBUTING.md)，Fork 仓库并提交 PR。
