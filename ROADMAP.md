## 🐛 已知问题

### 🔴 高优先级

- [ ] **[BUG-001]** PostgreSQL 连接池缓存键不一致导致数据库切换问题  
  *影响：用户切换数据库时可能查询错误的数据库*  
  *位置：`pool.go:35`, `schema.go:45`*  
  *优先级：P0*  
  *状态：待修复*

- [ ] **[BUG-002]** 缺少查询超时控制，长时间查询可能导致界面卡死  
  *影响：用户体验，可能需要强制关闭应用*  
  *位置：`query.go:10-97`*  
  *优先级：P0*  
  *状态：待修复*

- [ ] **[BUG-003]** 配置文件错误时静默失败，缺少用户提示  
  *影响：用户无法感知配置加载失败*  
  *位置：`config.go:65-85`*  
  *优先级：P1*  
  *状态：待修复*

---

### 🟡 中优先级

- [ ] **[BUG-004]** Redis 数据库数量硬编码为16，无法动态获取  
  *影响：某些 Redis 配置可能有不同的数据库数量*  
  *位置：`db/redis.go:120-125`*  
  *优先级：P2*  
  *状态：待优化*

- [ ] **[BUG-005]** 连接池无上限，可能导致资源耗尽  
  *影响：大量连接时内存占用过高*  
  *位置：`pool.go:46-59`*  
  *优先级：P1*  
  *状态：待优化*

- [ ] **[BUG-006]** 表结构查询结果未缓存，频繁查询性能差  
  *影响：重复查询表结构时响应慢*  
  *位置：`schema.go:169-187`*  
  *优先级：P2*  
  *状态：待优化*

- [ ] **[BUG-007]** Monaco Editor 在大数据集渲染时性能下降  
  *影响：查询结果超过1000行时界面卡顿*  
  *位置：前端代码*  
  *优先级：P1*  
  *状态：待优化*

---

### 🟢 低优先级

- [ ] **[BUG-008]** 日志文件未实现，调试困难  
  *影响：问题排查效率低*  
  *优先级：P3*  
  *状态：待开发*

- [ ] **[BUG-009]** 缺少单元测试的模块需要补充测试用例  
  *影响：代码质量保障*  
  *优先级：P3*  
  *状态：待补充*
